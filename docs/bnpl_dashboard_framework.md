# BNPL 业务看板框架

## 背景与目标

建设一个 HTML 版 BNPL 业务经营看板，参考现有飞书 DPL 看板的信息密度，以及业务 KPI 看板的颗粒度，采用黑色金融科技风格。

看板需要覆盖 BNPL 业务从申请、授信、放款、还款、逾期、收入到明细下钻的全链路经营指标。

## 产品与币种口径

BNPL 产品范围：

| 产品 ID | 分组 | 币种 |
| --- | --- | --- |
| 153 | 人民币 BNPL | CNY |
| 157 | 美金 BNPL | USD |
| 187 | 美金 BNPL | USD |

金额总览统一使用人民币口径：

- `product_id = 153` 使用人民币金额。
- `product_id in (157, 187)` 使用数仓中已换算为人民币的金额字段。
- 前端不临时换汇，不手工指定固定汇率。
- 如 SQL 探索中找不到已折人民币字段，再探索数仓汇率或金额宽表口径。
- 美金 BNPL 明细页可同时展示 USD 原币金额和 RMB 折算金额。

## 品牌命名口径

| 日期范围 | 展示名称 |
| --- | --- |
| 2026-01-01 之前 | 豆呗 |
| 2026-01-01 及之后 | 豆分期 |

看板标题建议为：`BNPL / 豆分期经营看板`。

筛选器中保留业务名称口径：

- 全部
- 豆呗：`date < '2026-01-01'`
- 豆分期：`date >= '2026-01-01'`

## 全局筛选

- 时间范围：最近 7 天、30 天、90 天、自定义。
- 时间颗粒度：日、周、月、季度、年。
- 产品分组：全部 BNPL、人民币 BNPL、美金 BNPL。
- 产品 ID：153、157、187。
- 业务名称：全部、豆呗、豆分期。
- 候选维度：国家、站点、平台、渠道、商户、店铺，字段待数据探索验证。
- 指标口径：申请口径、授信口径、放款口径、还款口径。

## 页面模块

### 1. 经营总览

核心 KPI：

- 申请数
- 授信通过数
- 放款订单数
- 放款金额，人民币口径
- 在贷余额，人民币口径
- 应还金额，人民币口径
- 实还金额，人民币口径
- 逾期金额，人民币口径
- 审批通过率
- 放款转化率
- 还款率
- 逾期率

展示要求：

- 总 BNPL、人民币 BNPL、美金 BNPL 分组展示。
- 金额类指标统一人民币口径。
- 支持同比、环比、趋势线和异常高亮。

### 2. BNPL 业务漏斗

漏斗链路：

用户进入 -> 申请 -> 授信 -> Offer -> 放款 -> 首还 -> 完结

每一步展示：

- 人数
- 订单数
- 金额
- 转化率

支持按产品分组、产品 ID、渠道、商户拆解。

### 3. 贷款与放款分析

指标：

- 放款金额趋势
- 放款订单趋势
- 平均单笔金额
- 产品 153、157、187 对比
- 人民币 BNPL 与美金 BNPL 对比
- 商户、店铺、渠道 Top 排名
- 放款失败、拒绝、取消等状态分布，字段待验证

### 4. 还款与资产质量

原始数据源：

| 人民币 / 豆呗豆分期 | `RItMw6psoiIWiTkgiqKcSdNrnwf` | 总表、月回款率及逾期率、当前逾期率、历史逾期率 |
| 美金 / 豆呗豆分期 | `YsxEw4TBRizTZGkpqRUcHGEvnOf` | 回款明细、月回款率及逾期率、当前逾期率 |

指标：

- 应还金额
- 实还金额
- 未还金额
- 提前还款
- 逾期金额
- DPD 1+、7+、30+、60+、90+
- 逾期率
- 回收率
- Vintage / Cohort 表现

### 5. 收入与费用

指标候选：

- 服务费
- 利息
- 手续费
- 罚息
- 收入趋势
- 产品收入贡献
- 商户收入贡献
- 收入 / 放款金额
- 收入 / 在贷余额

字段和口径待数据探索验证。

### 6. 用户与商户画像

指标候选：

- 新用户
- 老用户
- 复借用户
- 活跃用户
- 高价值商户
- 高风险商户
- 商户/店铺维度申请、放款、逾期、收入贡献

字段和口径待数据探索验证。

### 7. 底层数据明细

所有看板数据点需要支持点击下钻：

- KPI 卡片点击后打开明细抽屉。
- 趋势图点位点击后展示对应日期/周期明细。
- 排名表行点击后展示对应商户、店铺、产品或渠道明细。
- 明细展示对应 SQL 或飞书来源口径、筛选条件、字段列表。
- 支持搜索、排序和导出 CSV。

明细字段候选：

- 日期
- 产品 ID
- 产品分组
- 业务名称
- 用户 ID
- 商户 / 店铺
- 贷款单号
- 申请状态
- 授信状态
- 放款金额
- 放款金额人民币口径
- 应还金额
- 实还金额
- 逾期金额
- 逾期天数
- 费用收入

## 设计风格

- 黑色金融科技风。
- 深黑背景，深灰面板。
- 青绿色、蓝色作为主要数据高亮。
- 金色仅用于收入、利润等财务重点指标。
- 信息密度高，适合经营复盘和日常监控。
- 避免营销页式大 Hero，首屏直接进入可用看板。

## 标准开发流程

1. 需求澄清。
2. 数据探索：使用 MCP 探索候选库、候选表、字段和样例。
3. SQL 验证：编写 SQL 并通过 MCP 验证结果。
4. 前端开发：开发 HTML 看板。
5. 数据验证：对比页面展示数据和 SQL/飞书结果。
6. 持续优化：根据反馈调整口径、布局和交互。

## 当前数据探索原则

- 优先使用 DWS、ADS、DIM 层作为看板数据底座。
- 不跳过宽表直接查 ODS。
- SQL 前先确认候选表和字段。
- 还款与资产质量优先使用飞书多维表格原始数据。
- 每个指标必须保留可下钻到底层数据的来源口径。
